2. DETAILED PHASE-BY-PHASE WORKFLOW
================================================================================

--------------------------------------------------------------------------------
PHASE 1: LANDING & THEME SELECTION (`index.html`)
--------------------------------------------------------------------------------
- User Accesses: `index.html` (or root route).
- Trigger: Page initializes GSAP animations, particle effects, and reads saved theme.
- Actions Available:
  1. Theme Switcher: User clicks theme toggle pills (Burger / Pizza / Fanta).
     - System updates `:root` CSS design tokens instantly.
     - Saves theme state in `localStorage.setItem('cravego_theme', theme)`.
  2. Hero CTA: User clicks "Order Now" -> Navigates to `restaurant.html`.
  3. Navbar Navigation: Quick links to Menu, Cart, Orders, Profile, or Login.

--------------------------------------------------------------------------------
PHASE 2: RESTAURANT & MENU SELECTION (`restaurant.html` & `restaurant-menu.html`)
--------------------------------------------------------------------------------
- User Browses: Catalog of available food vendors (`restaurant.html`).
- Category Filter: Users filter restaurants by cuisine (Burger, Pizza, Beverages, Desserts).
- Action: User clicks a restaurant card -> Navigates to `restaurant-menu.html?id=X`.
- Menu View (`restaurant-menu.html`):
  - Fetches dish catalog associated with selected restaurant ID.
  - User clicks "+ Add to Cart" or adjusts dish quantities.
  - State Sync: Updates `localStorage.setItem('cravego_cart', JSON.stringify(cartItems))`.
  - UI Sync: Badge counter on Floating Cart icon increments in real time.

--------------------------------------------------------------------------------
PHASE 3: CART MANAGEMENT & DISCOUNTS (`cart.html`)
--------------------------------------------------------------------------------
- User Accesses: `cart.html`.
- Actions Available:
  1. Quantity Adjustment: Increase (`+`) or decrease (`-`) dish quantities or remove items.
  2. Order Summary Calculation:
     - Subtotal = Σ (Item Price × Quantity)
     - Delivery Fee & Tax calculated dynamically.
  3. Promo Code Engine: User types discount coupon (e.g. `SIVA95` or `CRAVE50`).
     - System verifies code validity -> Applies discount percentage/amount.
     - Displays updated bill breakdown and total savings.
  4. Proceed CTA: User clicks "Proceed to Checkout".

--------------------------------------------------------------------------------
PHASE 4: ROUTE GUARD & AUTHENTICATION (`login.html`)
--------------------------------------------------------------------------------
- Interceptor logic checks if user profile session (`localStorage.getItem('profile')`) exists:
  - IF Session Exists: Directly forwards user to `checkout.html`.
  - IF Session Missing:
    1. Triggers Toast Notification: "Please log in to proceed with your order".
    2. Redirects to `login.html?redirect=checkout.html`.
- User Options on `login.html`:
  - Existing User: Enters Username/Email & Password -> Click "Sign In".
  - New User: Switches tab to "Sign Up" -> Enters Name, Email, Phone, City & Password.
- Post-Authentication:
  - Saves active user profile into session memory (`profile`).
  - Automatically redirects user back to `checkout.html`.

--------------------------------------------------------------------------------
PHASE 5: CHECKOUT & PAYMENT PROCESSING (`checkout.html` & `payment.html`)
--------------------------------------------------------------------------------
- Checkout View (`checkout.html`):
  - Auto-populates delivery contact info from active logged-in user profile.
  - Collects Delivery Address, Street Name, Pincode, and Special Delivery Notes.
  - Displays Final Order Summary (Items, Discount, Grand Total).
  - User clicks "Proceed to Payment" -> Redirects to `payment.html`.
- Payment View (`payment.html`):
  - User selects preferred payment method:
    * UPI (Google Pay, PhonePe, Paytm, BHIM)
    * Credit / Debit Card (Card number, Expiry, CVV)
    * Net Banking (Popular banks dropdown)
    * Cash on Delivery (COD)
  - User clicks "Pay & Place Order":
    - Triggers simulated payment gateway validation.
    - Displays processing modal animation.

--------------------------------------------------------------------------------
PHASE 6: ORDER CONFIRMATION & RECEIPT GENERATION (`order-success.html`)
--------------------------------------------------------------------------------
- Payment Successful Event Triggered:
  - System generates unique Order ID (e.g., `#CRV-948201`).
  - Constructs order record combining:
    * Order ID & Timestamp
    * User Email & Address
    * Purchased Items List & Total Paid Amount
    * Payment Method & Transaction Status ("CONFIRMED")
  - Appends order record into user-isolated storage (`order_history`).
  - Clears active cart (`cravego_cart`).
- View Display (`order-success.html`):
  - Displays animated checkmark success icon.
  - Shows complete downloadable/printable Order Invoice.
  - Provides CTA buttons: "Track Order" (`order-history.html`) or "Back to Home".

--------------------------------------------------------------------------------
PHASE 7: USER PROFILE & ORDER HISTORY TRACKING (`profile.html` & `order-history.html`)
--------------------------------------------------------------------------------
- Order History View (`order-history.html`):
  - Reads `activeProfile.email`.
  - Filters stored orders matching current user email (Strict Privacy Isolation).
  - Renders interactive order history cards showing status progress:
    [Order Placed] -> [Kitchen Preparing] -> [Out for Delivery] -> [Delivered].
  - Allows user to click "Reorder" to reload items into cart.
- Profile View (`profile.html`):
  - Displays user profile avatar, name, email, phone, and saved delivery addresses.
  - Displays account statistics (Total Orders Placed, Total Savings).
  - Logout Button: Clicking Logout clears active profile session (`localStorage.removeItem('profile')`), 
    resets session, and safely redirects to homepage without wiping registered user databases.

3. ASYNCHRONOUS & AUXILIARY WORKFLOWS
================================================================================

--------------------------------------------------------------------------------
AI FOOD ASSISTANT CHATBOT WORKFLOW (`chatbot.js`)
--------------------------------------------------------------------------------
1. User clicks Floating Chatbot Widget on bottom right.
2. Chat window slides up displaying welcome greeting & Quick Prompt Pills:
   - "Track My Order"
   - "Available Promo Codes"
   - "Delivery Timings"
   - "Developer Info"
3. User types a prompt or selects a quick pill.
4. Intent Matching Engine evaluates query against keyword patterns:
   - Returns contextual answer, coupon code details, or order status.
5. Handles fallback responses gracefully for out-of-domain queries.

--------------------------------------------------------------------------------
NON-BLOCKING TOAST NOTIFICATION WORKFLOW (`script.js`)
--------------------------------------------------------------------------------
1. System or user action triggers notification (e.g., `alert("Item added to cart")`).
2. Interceptor overrides native browser `window.alert`.
3. Analyzes text message for classification:
   - Contains 'error' / 'failed' / 'required' -> Type: ERROR (Red Theme, Error Icon)
   - Contains 'success' / 'added' / 'placed' -> Type: SUCCESS (Green Theme, Check Icon)
   - Default -> Type: INFO (Amber Theme, Info Icon)
4. Dynamically appends glassmorphic Toast element into top-right viewport DOM.
5. Runs entry slide animation -> Waits 3000ms -> Runs exit fade animation -> Removes DOM element.

 |  2. Instantiates UserDAOImpl.                                              |
 +-------------------------------------+--------------------------------------+
                                       │
                                       ▼
 +----------------------------------------------------------------------------+
 | UserDAOImpl (com.cravego.dao.impl.UserDAOImpl)                             |
 |  1. Calls DBConnection.getConnection() to acquire JDBC connection.        |
 |  2. Prepares SQL Query:                                                    |
 |     "SELECT * FROM USERS WHERE email = ? AND password = ?"                 |
 |  3. Executes PreparedStatement with parameters (SQL Injection Safe).       |
 +-------------------------------------+--------------------------------------+
                                       │
                                       ▼
 +----------------------------------------------------------------------------+
 | MySQL Database                                                             |
 |  - Executes query against USERS table.                                     |
 |  - Returns ResultSet matching record.                                      |
 +-------------------------------------+--------------------------------------+
                                       │
                                       ▼
 +----------------------------------------------------------------------------+
 | UserDAOImpl & LoginServlet Output Processing                                |
 |  - Maps ResultSet into User POJO (com.cravego.model.User).                 |

 |  - Sends HTTP Redirect / JSON Response to Client.                          |
