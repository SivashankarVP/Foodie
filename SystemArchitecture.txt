1. ARCHITECTURAL OVERVIEW
================================================================================
CraveGo is structured as an enterprise-grade multi-tier web application following
the classic Model-View-Controller (MVC) pattern combined with the Data Access 
Object (DAO) architectural pattern.

The platform provides a dual-mode operational capabilities:
1. Client-Side Interactive Web Application (Single-Page App dynamics with multi-page HTML5 routing)
2. Enterprise Java Server Backend (Servlets + JDBC + MySQL Database)

--------------------------------------------------------------------------------
Key Architecture Principles:
- Separation of Concerns (SoC): Clear isolation between Presentation (HTML/CSS/JS),
  Control/Business Logic (Servlets/JS Controllers), and Data Access (DAO/JDBC/MySQL).
- Modular Design Token System: CSS variables powering dynamic multi-theme switches 
  (Burger, Pizza, Fanta modes) and Glassmorphism design system.
- User Isolation & Data Security: Multi-tenant client storage isolation combined with 
  relational foreign-key boundaries in MySQL.
- Non-Blocking Asynchronous Operations: Event-driven notification interceptor 
  and AI Chatbot engine.

2. TECHNOLOGY STACK
================================================================================
[FRONTEND LAYER]

[DATABASE LAYER]
- RDBMS: MySQL 8.0+
- Connection Protocol: MySQL Connector/J JDBC Driver

4. FRONTEND SUBSYSTEMS & MODULES
================================================================================
1. Multi-Theme Switching Engine (`script.js`):
   - Dynamically toggles CSS root variable definitions between Burger (Orange/Amber), 
     Pizza (Red/Gold), and Fanta (Deep Orange/Cyan) aesthetics.
   - Preserves theme state in `localStorage.setItem('cravego_theme', theme)`.

2. Glassmorphism Design System (`style.css`):
   - Built with frosted glass cards (`backdrop-filter: blur(12px)`), ambient drop shadows, 
     translucent borders, and floating depth layers.

3. Non-Blocking Notification Interceptor (`script.js`):
   - Overrides `window.alert` to route messages into animated Toast DOM elements.
   - Implements auto-dismiss timing, icon indicators, and non-blocking user workflow.

4. Native AI Assistant Chatbot (`chatbot.js` / `chatbot.css`):
   - Pattern-matching NLP engine analyzing user prompts against intent modules 
     (OrderStatus, PromoCodes, DeveloperInfo, DeliveryPolicy, MenuInfo).
   - Embedded directly into the ordering funnel with quick action pills.

5. Authentication Guard & Navigation Manager:
   - Restricts checkout (`checkout.html`) and history (`order-history.html`) pages.
   - Redirects unauthenticated users to `login.html` and restores routing state post-login.

│   ├── User.java             - Data representation for user account credentials & details.
│   ├── Restaurant.java       - Data representation for restaurant catalog metadata.
│   ├── Menu.java             - Data model for individual dish items and pricing.
│   ├── Order.java            - Header record model for user purchase transactions.

│   ├── RestaurantDAO.java    - Contract interface for retrieving restaurant listings.

│   ├── OrderDAO.java         - Contract interface for creating & querying orders.
│   └── OrderItemDAO.java     - Contract interface for order item insertion & retrieval.
│
├── com.cravego.dao.impl/
│   ├── UserDAOImpl.java      - JDBC SQL implementation of UserDAO.
│   ├── RestaurantDAOImpl.java- JDBC SQL implementation of RestaurantDAO.
│   ├── MenuDAOImpl.java      - JDBC SQL implementation of MenuDAO.
│   ├── OrderDAOImpl.java     - JDBC SQL implementation of OrderDAO.
│   └── OrderItemDAOImpl.java - JDBC SQL implementation of OrderItemDAO.
│
├── com.cravego.servlet/
│   ├── LoginServlet.java     - Handles POST login requests & session token creation.
│   ├── RegisterServlet.java  - Validates & creates new user accounts.
│   ├── RestaurantServlet.java- Serves JSON/HTML lists of available restaurants.
│   ├── MenuServlet.java      - Fetches dishes filtered by restaurant ID.
│   ├── OrderServlet.java     - Processes checkout submission & order creation.
│   └── OrderHistoryServlet.java - Returns user-isolated order history records.
│
└── com.cravego.util/
    └── DBConnection.java     - Centralized JDBC connection establishment utility.

7. END-TO-END DATA FLOW & LIFECYCLE
================================================================================
1. Restaurant & Dish Selection:
   User browses `restaurant.html` -> Selects vendor -> Redirected to `restaurant-menu.html` ->
   Selects food items -> JavaScript populates cart state in LocalStorage (`cravego_cart`).

2. Promo Code & Cart Calculation:
   User visits `cart.html` -> Modifies item quantities -> Enters promo code (e.g. `SIVA95`) ->
   Discount algorithm calculates subtotal, taxes, and final total.

3. Authentication Check:
   User clicks "Proceed to Checkout" -> Auth Guard verifies `activeProfile` ->
   If missing, redirects to `login.html` with return URL query param -> User logs in ->
   Returned to `checkout.html`.

4. Order Placement & Dispatch:
   User inputs delivery address & selects payment mode -> Order payload generated ->
   Dispatched to backend `OrderServlet` / saved in LocalStorage `order_history` ->
   Navigated to `order-success.html` displaying formatted receipt & Order ID.

5. Order Isolation & Tracking:
   User visits `order-history.html` or `profile.html` -> System queries orders matching 
   `activeProfile.email` -> Renders isolated user transaction logs.

================================================================================
8. SECURITY & DATA PROTECTION INITIATIVES
================================================================================
- PreparedStatements in JDBC: Eliminates SQL Injection risks across all DAO implementations.
- User Isolation Boundaries: Strict client-side and database filter parameters ensuring 
  users cannot view or alter peer order data.
- Controlled Session Teardown: Logout destroys active session tokens (`localStorage.removeItem('profile')`)
  while preserving registered user databases for subsequent sign-ins.
