SLIDE 1: Project Intro
--------------------------------------------------------------------------------
• Project: CraveGo - Next-Gen Interactive Food Ordering Platform
• Overview: CraveGo is an enhanced, high-performance web application designed to deliver an immersive culinary ordering experience. It combines a sleek glassmorphic UI with dynamic multi-theme switching, persistent multi-user session management, custom non-blocking notification systems, an AI-powered assistant chatbot, and a full Java (JDBC/Servlet) backend structure.

SLIDE 2: Tech Stack
--------------------------------------------------------------------------------

• Frontend (UI/UX):
  - Languages: HTML5, CSS3 (Vanilla CSS with Glassmorphic Design System)
  - JavaScript: ES6+ (DOM Manipulation, Web Storage API, Event Delegation)
  - Animations & Icons: GSAP (GreenSock Animation Platform), RemixIcon, Particles.js

• Backend (Server & Controllers):
  - Language: Java (JDK 17+)
  - Controller Layer: Java Servlets (javax.servlet / Jakarta Web APIs)
  - Data Access Layer: DAO Pattern with JDBC (Java Database Connectivity)
  - Build Tool: Apache Maven (pom.xml)

• Database:
  - RDBMS: MySQL 8.0+ (Relational Schema & Foreign Key Constraints)

SLIDE 6: Key Project Differentiators & Standout Features (10 Unique Implementations)
--------------------------------------------------------------------------------
1. Dynamic Multi-Theme Food Experience ⭐
   - Concept: Instant theme morphing between Burger, Pizza, and Fanta modes with synchronized UI colors, backgrounds, hero elements, and brand identities.
   - Differentiator: Unlike typical light/dark toggles, theme changes dynamically re-skin the entire interface while maintaining brand consistency across all pages.

2. Custom Glassmorphism Design System
   - Concept: Built a bespoke, premium UI style guide from scratch without relying on generic Bootstrap templates.
   - Highlights: Frosted glass cards (`backdrop-filter: blur`), ambient lighting effects, layered depth with translucent borders, and rounded modern components.

3. Integrated AI Food Assistant Chatbot
   - Concept: Native assistant embedded directly inside the food ordering funnel rather than a generic floating widget.
   - Capabilities: Handles delivery policies, live order status checks, promo coupon queries (e.g., SIVA95), restaurant details, and developer bio; features quick action buttons and suggested replies.

4. Non-Blocking Custom Notification System
   - Concept: Replaced disruptive native browser popups with an animated glassmorphic Toast notification system (`window.alert()` global interceptor).
   - Benefits: Smooth CSS entry/exit animations, non-blocking user flow, and distinct visual feedback states (Success, Warning, Error).

5. User-Isolated Local Storage Architecture
   - Concept: Developed a multi-tenant client-side storage model simulating true enterprise multi-user security.
   - Highlights: Account-isolated order histories, individual user profiles, session state persistence, and clean logout teardown that retains registered user credentials.

6. Login Guard Navigation Protection
   - Concept: Implemented route protection logic enforcing authentication before accessing restricted pages.
   - Protected Flow: Cart → Checkout → [Guard: Not Logged In?] → Login Page → Auto-Return to Checkout.

7. Smooth Interactive UI & GSAP Micro-Animations
   - Concept: Focus on high visual polish using GreenSock Animation Platform (GSAP) and CSS micro-interactions.
   - Highlights: Dynamic hover responses, smooth page entry transitions, interactive hero section, animated CTA buttons, and fluid scroll effects.

8. Industry-Standard Restaurant Ordering Flow
   - Concept: Implemented a realistic, multi-step food delivery workflow reflecting real-world commercial platforms (Swiggy/Zomato).
   - Lifecycle: Restaurant Selection → Menu Customization → Cart & Discounts → Checkout Address → Payment Processing → Order Confirmation → User Order Tracking.

9. Layered Java Backend Architecture (Servlet + DAO + JDBC)
   - Concept: Structured Java backend adhering to enterprise MVC and Data Access Object (DAO) software engineering patterns.
   - Pipeline: Servlet (Controller) → DAO (Business/Data Interface) → JDBC (Driver & Connection Pool) → MySQL Relational DB.

10. Modular & Reusable Component Architecture
    - Concept: Clean separation of concerns across JavaScript ES6+ modules and centralized CSS variable design tokens.
    - Benefits: Highly maintainable, clean code structure enabling effortless scaling, component reusability, and maintainability.

SLIDE 7: Important Algorithms & Technical Logic
--------------------------------------------------------------------------------
1. Global `window.alert()` Interceptor:
   - Overrode native `window.alert` in `script.js` to parse message intent ('error' vs 'success') and trigger animated DOM Toast overlays, eliminating disruptive native browser popups.

2. Intent Matching Engine (Chatbot):
   - Developed a lightweight regex matching function `match(keywords)` in `chatbot.js` that evaluates user inputs against intent dictionaries (Developer Info, Coupons, Order Tracking) with fallback constraints to maintain app-specific domain context.

3. Page-Aware Navigation State Switcher:
   - Evaluated `window.location.pathname` inside `switchTheme()` to dynamically prefix hash routes (`index.html#home` vs `#home`), ensuring seamless cross-page navigation from subpages back to the homepage.

4. User-Isolated Order Filter:
   - Linked each newly placed order with `userEmail` in local storage, and dynamically filtered `order_history` against `activeProfile.email` to preserve strict privacy per user account.

--------------------------------------------------------------------------------
SLIDE 8: Major Challenge Faced & Solution
--------------------------------------------------------------------------------
• Challenge: 
  When logging out, calling `localStorage.clear()` wiped out all registered user accounts (`users`) and order history stored on the browser, preventing returning users from logging back into previously registered accounts. Additionally, navigation links on subpages were losing their `index.html` prefix when theme switching executed.

• Solution:
  1. Refactored authentication tear-down to target `localStorage.removeItem('profile')` instead of clearing the entire Web Storage database, ensuring user credentials remain persistently saved.
  2. Implemented route prefix checking in `script.js` to distinguish between the homepage and subpages before re-writing anchor `href` attributes.
