SLIDE 1: Project Intro
--------------------------------------------------------------------------------
• Project: Foodie - Next-Gen Interactive Food Ordering Platform
• Overview: Foodie is an enhanced, high-performance web application designed to deliver an immersive culinary ordering experience. It combines a sleek glassmorphic UI with dynamic multi-theme switching, persistent multi-user session management, custom non-blocking notification systems, an AI-powered assistant chatbot, and a full Java (JDBC/Servlet) backend structure.

SLIDE 2: Tech Stack
--------------------------------------------------------------------------------
• Frontend (UI/UX):
  - Languages: HTML5, CSS3 (Vanilla CSS with Glassmorphic Design System)
  - JavaScript: ES6+ (DOM Manipulation, Web Storage API, Event Delegation)
  - Animations & Icons: GSAP (GreenSock Animation Platform), RemixIcon, Particles.js

• Backend (Server & Controllers):
  - Language: Java (JDK 17+)
  - Controller Layer: Java Servlets (javax.servlet / Jakarta Web APIs)
  - Data Access Layer: DAO Pattern with JDBC (Java Database Connectivity)
  - Build Tool: Apache Maven (pom.xml)

• Database:
  - RDBMS: MySQL 8.0+ (Relational Schema & Foreign Key Constraints)

SLIDE 5: Page Screenshots & User Flow
--------------------------------------------------------------------------------
Order Flow Sequence:
1. Home Page (index.html) -> Interactive hero section with theme switcher.
2. Restaurant Menu (restaurant-menu.html) -> Browse dishes, filter by category, add to cart.
3. Shopping Cart (cart.html) -> Modify quantities, apply promo codes (e.g. SIVA95).
4. Authentication (login.html) -> Enforced login/signup before checkout.
5. Checkout & Payment (checkout.html & payment.html) -> Enter address, select payment (UPI/Card).
6. Order Confirmation (order-success.html) -> Invoice preview & order ID generation.
7. User Profile & Order History (profile.html & order-history.html) -> View saved user details and user-isolated past orders.

SLIDE 6: New Features You Added (Unique Additions)
--------------------------------------------------------------------------------
• Interactive AI Assistant Chatbot: Responsive full-screen mobile chatbot with intent-based pattern matching, quick action replies, and developer info integration.
• Non-Blocking Glassmorphic Toast System: Globally overridden browser `window.alert()` with smooth animated CSS toasts.
• Dynamic Multi-Theme Switcher: Instant theme morphing between Burger, Fanta, and Pizza modes with synchronized UI elements and color schemes.
• User-Isolated Data Architecture: Multi-account signup/signin persistence in Web Storage with user-specific order history filtering.
• Login Guard Security: Automatic auth redirection when attempting checkout without a logged-in session.

--------------------------------------------------------------------------------
SLIDE 7: Important Algorithms & Technical Logic
--------------------------------------------------------------------------------
1. Global `window.alert()` Interceptor:
   - Overrode native `window.alert` in `script.js` to parse message intent ('error' vs 'success') and trigger animated DOM Toast overlays, eliminating disruptive native browser popups.

2. Intent Matching Engine (Chatbot):
   - Developed a lightweight regex matching function `match(keywords)` in `chatbot.js` that evaluates user inputs against intent dictionaries (Developer Info, Coupons, Order Tracking) with fallback constraints to maintain app-specific domain context.

3. Page-Aware Navigation State Switcher:
   - Evaluated `window.location.pathname` inside `switchTheme()` to dynamically prefix hash routes (`index.html#home` vs `#home`), ensuring seamless cross-page navigation from subpages back to the homepage.

4. User-Isolated Order Filter:
   - Linked each newly placed order with `userEmail` in local storage, and dynamically filtered `order_history` against `activeProfile.email` to preserve strict privacy per user account.

--------------------------------------------------------------------------------
SLIDE 8: Major Challenge Faced & Solution
--------------------------------------------------------------------------------
• Challenge: 
  When logging out, calling `localStorage.clear()` wiped out all registered user accounts (`users`) and order history stored on the browser, preventing returning users from logging back into previously registered accounts. Additionally, navigation links on subpages were losing their `index.html` prefix when theme switching executed.

• Solution:
  1. Refactored authentication tear-down to target `localStorage.removeItem('profile')` instead of clearing the entire Web Storage database, ensuring user credentials remain persistently saved.
  2. Implemented route prefix checking in `script.js` to distinguish between the homepage and subpages before re-writing anchor `href` attributes.
